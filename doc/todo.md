### 高優先級 (High) - 後端核心 API 與資料庫

#### [x] 1. 資料庫建置與 SQLAlchemy Model 定義
- **狀態**: Done
- **相依關係**: 無
- **驗收條件**: 
  1. 使用 SQLAlchemy 定義 `users`, `merchants`, `products`, `orders`, `order_items` 表。
  2. 欄位包含外送員配單必備的 `driver_status` 與 `last_idle_at`。
  3. 完成本地 SQLite 初始遷移並成功建立資料庫檔案。

#### [x] 2. 用戶註冊與 JWT 登入 API
- **狀態**: Done
- **相依關係**: 項目 1
- **驗收條件**: 
  1. 實作 `POST /api/auth/register`，支援註冊消費者、店家與外送員。密碼須進行雜湊加密。
  2. 實作 `POST /api/auth/login`，驗證成功後核發 JWT Token。
  3. 實作 API 路由角色權限檢查依賴項。

#### [x] 3. 商家菜單管理與瀏覽 API
- **狀態**: Done
- **相依關係**: 項目 2
- **驗修條件**: 
  1. 實作商家商品管理：新增、修改、軟刪除（將 `is_available` 設為 `False`）餐點。限制僅該商家帳戶可操作。
  2. 實作消費者瀏覽商家列表與瀏覽特定商家菜單的 API。

#### [x] 4. 消費者點餐與下單 API
- **狀態**: Done
- **相依關係**: 項目 3
- **驗收條件**: 
  1. 實作 `POST /api/orders`，消費者可送出購物車餐點。
  2. **限制驗證**：限制單次下單僅能點選單一商家的餐點，否則回傳錯誤。
  3. **金額計算**：固定外送費 39 元，無起送價限制，自動計算總金額並寫入資料庫。

#### [x] 5. 商家訂單狀態管理 API
- **狀態**: Done
- **相依關係**: 項目 4
- **驗收條件**: 
  1. 實作接單 API，使訂單狀態從 `PENDING_STORE` 變更為 `PREPARING`。
  2. 實作準備完成 API，將訂單狀態從 `PREPARING` 變更為 `READY_FOR_PICKUP`。

#### [x] 6. 自動派單系統與外送員配送 API
- **狀態**: Done
- **相依關係**: 項目 5
- **驗收條件**: 
  1. 當訂單變更為 `READY_FOR_PICKUP` 時，自動搜尋目前狀態為 `idle` 的外送員，並依照其 `last_idle_at` 遞增排序，將訂單指派給**空閒最久**的外送員。
  2. 指派後，訂單狀態改為 `DELIVERING`，外送員狀態改為 `delivering`。
  3. 實作外送員標記送達 API `POST /api/orders/{id}/complete`。完成後訂單變更為 `COMPLETED`，外送員狀態改回 `idle`，且其 `last_idle_at` 更新為目前時間。
