# 餐廳外送系統 - 需求規格書 (Software Requirement Specification)

## 1. 專案概述與目標 (Project Overview & Goal)
*   **首要目標**：建立一個多角色的「餐廳外送系統」，連接消費者、餐廳、外送員與管理者，提供流暢的餐點瀏覽、訂購、外送與管理體驗。
*   **目標受眾**：主要針對學生及上班族群，系統前端需優化行動端/響應式 (RWD) 的瀏覽體驗。

## 2. 核心功能與角色職責 (Core Features & Functional Requirements)

### 2.1 消費者 (Consumer)
*   **瀏覽餐廳與餐點**：可按餐廳分類或餐點內容進行瀏覽。
*   **購物車管理**：可將餐點加入購物車、修改數量、並計算小計與總額。
*   **建立訂單與付款**：
    *   **到店自取消費**：提供「刷卡」或「現金」兩種付款方式。
    *   **外送服務**：統一規定僅限使用「刷卡」付款。
*   **查看訂單狀態**：可即時追蹤訂單目前的進度狀態。

### 2.2 餐廳 (Restaurant)
*   **菜單管理**：提供餐點上架、下架、編輯價格、名稱與描述之功能。
*   **訂單管理**：能接收新訂單，並可進行確認接單或取消訂單。

### 2.3 外送員 (Delivery Agent)
*   **接單管理**：可於接單列表中「自由接單」。
*   **配送流程**：接單後可更新訂單狀態（如：已取餐、已送達）。

### 2.4 管理者 (Admin)
*   **營收統計**：可查看各個餐廳的營收狀況與統計數據。
*   **帳戶管理**：擁有平台帳戶（消費者、餐廳、外送員）的管理與權限控制權。

## 3. 技術選型與環境限制 (Technical Stack & Constraints)
*   **前端框架**：React
*   **後端框架**：FastAPI
*   **開發資料庫**：SQLite (本機開發環境)
*   **生產資料庫**：PostgreSQL (Render PostgreSQL 正式環境)
*   **部署平台**：Render.com

## 4. UI/UX 與設計偏好
*   考慮學生與上班族群多以手機操作，界面設計需以 Mobile-first 或 RWD 為主，提供簡潔明瞭的下單與狀態追蹤流程。

## 5. 待確認事項 (Pending Issues)
*   線上刷卡金流之模擬串接方式與金流業者選型。
*   外送費用的計算邏輯（如按距離、固定外送費等）。
*   外送員與餐廳的拆帳拆成邏輯。
