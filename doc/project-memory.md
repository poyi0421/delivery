## 1. 關鍵技術決策
- **前後端分離架構**：後端採用 FastAPI 提供 RESTful API，前端採用 React (Vite)。
- **資料庫策略**：
  - 本地開發環境：使用 SQLite (`delivery.db`)，以輕量、無需配置、檔案化為原則，加速開發迭代。
  - 正式部署環境：使用 Render PostgreSQL，提供穩定的正式資料儲存。
  - 所有資料表欄位設計皆透過 SQLAlchemy 進行對應，以確保 SQLite 與 PostgreSQL 語法之最大相容性。
- **部署平台**：Render.com。
  - 前端 React 託管於 Static Site 或由 FastAPI 後端代理。
  - 後端 FastAPI 部署為 Web Service，並建立 Render PostgreSQL 作為正式資料庫。
- **前端設計與主視覺色彩**：
  - 前端頁面維持**極簡潔風格**，但主視覺採用**明亮顯眼的安全防護顏色**（如高能見度的螢光安全橘 `#FF6F00` 或螢光綠 `#CCFF00`），藉此保護在馬路上奔波的外送員安全。

## 2. 業務規則與重要限制
- **外送員配單機制（空閒最長優先）**：
  - 本系統無 GPS 定位追蹤。
  - 系統採用自動指派。當訂單狀態為 `READY_FOR_PICKUP` (待取餐) 時，系統自動查詢目前狀態為 `idle` 的外送員，並依照其「成為空閒的起算時間 (`last_idle_at`)」進行排序，優先指派給**空閒時間最長**的外送員。
  - **外送員為常駐上線，不提供上下線或休息機制**（配送中除外，送達後即自動改回 `idle`）。
- **多商家限制**：
  - 消費者單次訂單只能選擇**同一家商店**的商品，不開放單次訂單跨商家點餐。
- **外送費與起送金額**：
  - 固定每筆訂單收取外送費 **39 元**。
  - 平台與店家**不設置起送價門檻限制**（點選任意金額即可下單）。
- **支付與取消限制**：
  - 僅支援**貨到付款 (COD)** 支付方式（外送員送達時收取現金）。
  - **完全不支援任何訂單取消與退款機制**，訂單流程一旦啟動就必須執行到送達並收取現金。
