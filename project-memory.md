## 💡 已確認的專案決策與技術選型 (Project Decisions & Tech Stack)
*   **前後端分離**：前端採用 React SPA 設計，後端採用 Python FastAPI 框架。
*   **資料庫隔離**：本地開發環境與單元測試使用 SQLite 資料庫（利於快速迭代），正式部署環境使用 Render PostgreSQL 資料庫。
*   **部署環境**：專案將部署於 Render.com 雲端平台。

---

## 🔒 關鍵資安決策
*   **密碼儲存**：一律使用 `bcrypt` 進行雜湊，不留明文密碼。
*   **API 授權**：基於 JWT (JSON Web Token) 做 Stateless 狀態驗證。
*   **憑證安全**：`.env` 不進入 Git，需提供 `.env.example`。

---

## 🎯 已確認的業務規則 (Confirmed Business Rules)
*   **四種角色權限**：系統存在消費者 (Consumer)、餐廳 (Restaurant)、外送員 (Delivery Agent) 與管理者 (Admin) 等四種角色。
*   **外送付款限制**：
    *   到店自取訂單：可使用「現金」或「刷卡」付款。
    *   外送訂單：強制且統一使用「刷卡」付款，不受理現金貨到付款。
*   **外送員自由接單**：外送員能在接單大廳中自由挑選訂單承接，不進行系統強制指派。
*   **歷史數據防護**：訂單明細（`order_items`）必須記錄交易發生時的餐點名稱與單價快照，不得直接依賴 `meals` 表的即時欄位，以防範日後菜單變動時歷史財務報表錯亂。
